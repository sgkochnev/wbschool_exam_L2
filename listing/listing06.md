Что выведет программа? Объяснить вывод программы. Рассказать про внутреннее устройство слайсов и что происходит при передачи их в качестве аргументов функции.

```go
package main

import (
	"fmt"
)

func main() {
	var s = []string{"1", "2", "3"}
	modifySlice(s)
	fmt.Println(s) // {"3", "2", "3"}
}

func modifySlice(i []string) {
	i[0] = "3"				// i = {"3", "2", "3"} len = 3 cap = 3
	i = append(i, "4") 		// i = {"3", "2", "3", "4"} - здесь уже новый слайс, так как произойдет расширение слайса. len = 4 cap = 6
	i[1] = "5" 				// i = {"3", "5", "3", "5"} len = 4 cap = 6 
	i = append(i, "6")      // i = {"3", "5", "3", "5", "6"} - тот же слайс, так как емкость позволяет добавлять элементы.  len = 5 cap = 6
}
```

Ответ:
```
["3", "2", "3"]

Слайс в представлении go это структура с тремя полями:
type sliceHeader struct {
	ptr unsafe.Pointer 	// указатель на массив
	len int				// длина
	cap int				// емкость
}

При передаче слайса в качестве аргументов функции происходит
копирование внутренней структуры слайса. Изменения вносимые в слайс
при обращении по индексу будут отображаться в исходном слайсе. (строка 17)

При вызове функции append происходит добавление элемента в слайс, (строка 18)
если емкость позволяет добавить элемент, иначе происходит расширение
слайса, т.е. создается новый массив (в 2 раза больше исходного, только для малых размеров) в него копируются все элементы
из исходного и в структуре `sliceHeader` меняется указатель на массив.

После такой опрерации все изменения будут происходить уже в новом слайсе.

Так как функция `modifySlice` не возвращает новый слайс вывод программы будут
["3", "2", "3"].
```
